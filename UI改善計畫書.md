# 介面改善計畫書 — 從「塑膠感」到「低飽和高質感」

## 一、現況診斷（實際操作 + 量測）

逐一檢視後，「塑膠感」來自這幾個具體來源，不是錯覺：

| # | 問題 | 實際證據 | 為何顯廉價 |
|---|---|---|---|
| 1 | **高飽和原色調色盤** | 日色全是 Tailwind 500 級：`#f97316`/`#10b981`/`#3b82f6`/`#ef4444`/`#ec4899`/`#14b8a6`… | 飽和度太高、太「螢光」，是塑膠感第一元兇 |
| 2 | **光澤高光（gloss）** | marker 用 `radial-gradient(circle at 30% 25%, rgba(255,255,255,.45)…)` | 字面意義上的「塑膠泡泡」反光 |
| 3 | **純白面板** | 日卡 `bg-white/95`、面板 `bg-slate-50` | 死白 + 鮮色 = 廉價對比 |
| 4 | **漸層過度使用** | 標頭、按鈕、日編號徽章、chip 全是 `linear-gradient(…color-mix…#0f172a)` | 到處漸層 = 雜、不高級 |
| 5 | **彩色發光陰影** | `box-shadow … color-mix(in srgb, 日色 40%, transparent)`；日卡 `shadow-xl`（20px/25px） | 螢光描邊、陰影過重 = 玩具感 |
| 6 | **emoji 當 UI** | 停靠點圖示、徽章（⚠️🍱✂️🌙☀️🚗）、統計（📅📍🚗） | emoji 跨平台不一致、視覺重量不齊、讀起來「可愛」而非「精緻」 |
| 7 | **字重過重** | 大量 `font-black`（900） | 整片都喊話 = 沒有層次、不優雅 |
| 8 | **裝飾性動態** | `aurora-text` 流動漸層字、斜紋紋理 `repeating-linear-gradient`、彈跳箭頭 | 炫技、增加視覺噪音 |

## 二、目標美學

> 極度美觀簡潔、低飽和、高質感、使用舒服、排佈合理。

設計語言：**北歐極簡 × 編輯級排版（editorial）**。少即是多——用「留白、層次、克制的色彩、紮實的字體系統」取代「漸層、光澤、鮮色、emoji」。

### 設計原則
1. **降飽和**：所有日色改為去飽和、降明度的「莫蘭迪／北歐」版本（飽和度約砍半），只在小面積（圓點、細線、標籤）出現，大面積用中性色。
2. **去光澤**：拿掉所有 radial 高光與斜紋紋理，改用平塗 + 極細邊框（1px hairline）+ 柔和中性陰影。
3. **紙感面板**：純白改為帶極淡暖灰的「紙色」（如 `#fafaf9` / `#f7f7f5`），面板邊框用 `1px rgba(0,0,0,0.06)` hairline 取代重陰影。
4. **字體層次**：少用 900；改以 size + 字色 + 字距建立層次（標題 600–700、內文 400–500、標籤 500）。Outfit 走標題與數字、Noto Sans TC 走中文內文。
5. **陰影系統**：統一兩級柔和中性陰影（`0 1px 2px rgba(0,0,0,.04)` / `0 8px 24px -8px rgba(15,23,42,.12)`），不再用彩色 glow。
6. **圖示系統**：見下方「待你決定」的圖示問題。

### 調色盤（提案，低飽和版）
- 中性骨架：`stone/zinc` 系——背景 `#0c1116`（地圖深底）、面板 `#fafaf9`、文字 `#2a2e35`/`#6b7280`、hairline `#e7e5e4`。
- 日色（去飽和後，僅舉例，實作會微調成和諧的一組）：
  D1 暮橙 `#c2774e`、D2 苔綠 `#5f8a6b`、D3 霧藍 `#5b7a99`、D4 薰衣草 `#8a7faa`、D5 赭黃 `#c2a15a`、D6 陶紅 `#b5685f`、D7 灰青 `#5f8a86`、D8 玫瑰灰 `#b07f93`、D9 靛灰 `#6a7196`、D10 石板 `#7c8694`。
  （統一以「明度 60–70 / 飽和度 25–35%」收斂，確保十色擺一起是和諧而非吵鬧。）

## 三、各畫面排佈調整

- **總覽（Overview）**：hero 卡縮小留白加大、拿掉 aurora 流動字（改靜態細緻標題）、統計改細線分隔的純文字（去 emoji）、日卡改紙色 + hairline + 去漸層條（改 2px 純色頂邊），日卡資訊密度下降一階。
- **單日時間軸（Timeline）**：標頭去斜紋、去漸層改平塗去飽和日色（或紙色 + 日色細節）；卡片去鮮色底，作用中用「左側 2px 日色 + 極淡日色底」標示；連接段（車程）字級縮小、加細線；徽章改細線描邊膠囊（pill）取代填色塊。
- **停靠點詳情（Stop Detail）**：標頭去紋理去大 emoji 浮水印，改乾淨色塊 + 小型分類標籤；動作列重排（見功能需求）。
- **清單／資訊頁**：卡片改紙色 + hairline；勾選態降彩度。
- **地圖 marker**：去光澤；改實心去飽和日色 + 白色細環 + 細陰影；作用中用單層柔和脈衝（降不透明度）。
- **TabBar／日別 chips**：玻璃感保留但降亮、去彩色陰影。

## 四、功能需求（你本次新增的兩項）

### A. 每個停靠點多一個「相關資訊」按鈕
- 資料層 `Stop` 介面新增 `infoUrl?: string` 與 `infoLabel?: string`。
- 詳情卡動作列從 2–3 顆重排為清楚的一排：**① 導航（Google Maps directions）｜② 在地圖開啟（Google Maps place）｜③ 相關資訊（infoUrl，顯示來源如「維基百科／官方網站」）｜④ 預約/官網（既有 booking/official，若有）**。
- 主行動（導航）維持最顯眼；其餘為次級樣式，視覺克制。
- 內容來源：由「內容擴充研究」工作流為每個點查一個權威連結（官方 > 維基 > 優質導覽），已於本次背景啟動。

### B. 每個停靠點說明更詳細且正確
- 由背景研究工作流（每天一個 agent、逐點查證）擴寫為 3–5 句、查證過的繁中說明，並回報任何事實更正。
- 完成後合併進 `itinerary.ts` 的 `desc`，並把研究發現的更正一併修掉。

## 五、實作階段

1. **設計 tokens**：在 `index.css` 建一組 CSS 變數（紙色、文字色、hairline、兩級陰影、去飽和日色表），全站改用 token。
2. **元件重構**：Overview / Timeline / StopDetail / Checklists / Info / TabBar / DayChips / markers 套用新 token，移除漸層/光澤/彩色陰影/多餘 emoji，調整排佈與字重。
3. **圖示系統**：依你選擇（見下）導入線性圖示或精煉 emoji。
4. **功能 A**：資料層加欄位 + 詳情卡動作列重排。
5. **功能 B**：合併研究結果（desc + infoUrl + 更正）。
6. **驗證**：桌機 + 手機逐頁檢視、build、推送。
